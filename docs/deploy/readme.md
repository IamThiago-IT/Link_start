# Deploy

## Frontend

- [Render](https://render.com/) — hospedagem para sites e apps estáticos/dinâmicos
- [Fly.io](https://fly.io/) — deploy de aplicações distribuídas
- [Railway](https://railway.app/) — deploy rápido para frontends e APIs
- [Koyeb](https://www.koyeb.com/) — plataforma para aplicações e serviços

## Backend

- [Supabase](https://supabase.com/) — backend como serviço (Postgres, auth, storage)
- [Neon](https://neon.tech/) — Postgres como serviço (DB)
- [MongoDB](https://www.mongodb.com/) — banco de dados NoSQL gerenciado
- [Firebase](https://firebase.google.com/) — BaaS (Realtime DB, Auth, Functions)
- [Railway](https://railway.app/) — deploy e gestão de serviços backend
- [Fly.io](https://fly.io/) — executar serviços de backend globalmente

## Fullstack

- [Supabase](https://supabase.com/) — oferece camada completa para apps fullstack
- [Firebase](https://firebase.google.com/) — ferramentas para apps front+back
- [Render](https://render.com/) — pode hospedar tanto front quanto back

## No-code / Low-code

- [Netlify](https://www.netlify.com/) — deploy contínuo para sites estáticos (no-code friendly)
- [Webflow](https://webflow.com/) — construtor visual e hosting
- [Bubble](https://bubble.io/) — plataforma no-code para apps web

---

Se quiser, posso ajustar as categorias (por exemplo mover `Neon` para "Fullstack") ou adicionar/remover provedores.
